# 修复 AI 输出不稳定问题的实施计划

用户反馈“还是有问题”，经过仔细审查，发现当前基于**数组索引** (`aiMsgIndex`) 更新消息的方式存在重大隐患。如果在 AI 生成过程中消息列表发生变化（如删除消息、加载历史），索引会失效，导致内容写入错误的对象或丢失。

此外，异常处理路径中缺少缓冲区刷新，可能导致出错前的最后一段内容丢失。

## 核心修复逻辑

### 1. 弃用索引，改用对象引用
*   **现状**：`messages.value[aiMsgIndex].content += ...`
*   **问题**：如果数组发生 `splice` 或 `shift`，`aiMsgIndex` 将指向错误的消息。
*   **修复**：在 `push` 后立即获取该消息的**响应式代理对象**，后续直接操作该对象。
    ```javascript
    messages.value.push(newMsg)
    const aiMessage = messages.value[messages.value.length - 1] // 获取引用
    // 后续直接操作 aiMessage.content += ...
    ```

### 2. 完善缓冲区刷新机制
*   在 `catch` 块（发生错误时）和 `done`（流结束时）都强制调用 `flushBuffer()`，确保毫秒级的数据也不会丢失。

### 3. 代码修改预览 (`Chat.vue`)
我们将重构 `sendMessageStream` 函数：

```javascript
const sendMessageStream = async (msg) => {
  // ... 初始化 ...
  
  // 1. 创建消息并获取引用
  const newMsg = { type: 'ai', ... }
  messages.value.push(newMsg)
  const aiMessage = messages.value[messages.value.length - 1] // 获取响应式引用
  
  // ... fetch ...

  const flushBuffer = () => {
    if (pendingContent) {
      aiMessage.content += pendingContent // 直接操作引用
      // ...
    }
  }

  try {
    while (true) {
      // ... 读取流 ...
      if (done) {
        flushBuffer() // 确保最后的内容被写入
        break
      }
      // ... 处理数据 ...
    }
  } catch (error) {
    flushBuffer() // 发生错误前的内容也要保留
    aiMessage.content += `\n[错误: ${error.message}]`
  } finally {
    aiMessage.isStreaming = false
    // ...
  }
}
```

## 预期效果
*   **极高的稳定性**：无论用户在生成过程中如何操作界面（删除其他消息、滚动等），AI 的回答都能准确写入对应的气泡中。
*   **零数据丢失**：即使网络中断，已接收到的最后一个字符也会被正确显示。
*   **持续流畅**：保留之前的数据缓冲机制，确保渲染性能。
